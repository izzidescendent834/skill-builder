What's a Skill?

A skill is a markdown file with instructions that your AI coding agent (Claude Code, Codex, Cursor) can execute as a slash command. Example: /pr-dashboard runs a script that checks all your repos for open PRs.

Skills live at ~/.claude/commands/ and are instantly available in your agent.

What's an MCP Server?

MCP (Model Context Protocol) lets your AI agent talk to external services — APIs, databases, SaaS tools. An MCP server exposes tools (actions the agent can take) and resources (data the agent can read).

Example: a Stripe MCP gives your agent list_charges, create_refund tools so it can manage payments directly.

How skill-builder works

skill-builder reads data from three sources on your machine and finds patterns worth automating:

1. Shell History

Reads ~/.zsh_history, ~/.bash_history, or fish history.

Finds: repeated commands, command sequences (A then B then C), and parameterizable patterns (same command with different arguments).

No setup needed. Works on any machine with a shell.

2. Desktop Telemetry

Reads the Cowork.ai SQLite database, or any compatible DB.

Finds: heavy app usage, frequently visited URLs, context switches between apps, and repeated keystrokes.

Optional. Install Cowork.ai for deeper analysis, or use the demo data generator.

3. Git History

Reads commit history from repos on your machine.

Finds: repeated commit message patterns, files that always change together (co-change analysis), and hot files that need frequent attention.

Automatic. Detects git repos in common locations.

Skills vs MCPs

Skills are instructions for your AI agent — markdown files that tell it how to do something. Think: recipes.

MCP servers are live connections between your agent and external services — they give the agent new tools. Think: power tools.

Use skills for workflows you repeat. Use MCPs for services you want your agent to access directly.

The SKILL.md Format

Skills use the SKILL.md standard (supported by 30+ platforms including Claude Code, Codex, Cursor, Gemini CLI):

---
name: my-skill
description: "When to trigger and what it does"
---

# My Skill

Instructions for the agent go here.
Bash commands, API calls, SQL queries — whatever gets the job done.

```bash
echo "This actually runs"
```
